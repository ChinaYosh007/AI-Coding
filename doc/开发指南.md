# 开发指南

## 1. 环境要求

- Java 21
- Node.js（建议使用当前 LTS 版本）与 npm
- MySQL 8
- Redis

后端由 Maven Wrapper 管理，无需预先安装 Maven。

## 2. 配置 `.env`（必做）

后端的 `application.yml` 通过 `spring-dotenv` 从 `backend/coding-server/.env` 读取数据库、Redis、模型服务、OSS 和图片搜索配置。配置文件不随代码提交，首次启动前执行：

```powershell
Copy-Item backend/coding-server/.env.example backend/coding-server/.env
```

然后编辑 `backend/coding-server/.env`，至少填写以下实际可用的配置组：

| 配置组 | 必填变量 | 用途 |
| --- | --- | --- |
| MySQL | `MYSQL_URL`、`MYSQL_PORT`、`MYSQL_DATABASE`、`MYSQL_ACCOUNT`、`MYSQL_PASSWORD` | 业务数据存储 |
| Redis | `REDIS_HOST`、`REDIS_PORT`、`REDIS_PASSWORD` | Session、缓存和 AI 记忆 |
| 模型服务 | `AGENT_URL`、模型名、`STREAM_THINKING_ENABLED` 及对应 API Key | 路由、代码生成与流式对话；DeepSeek V4 普通代码生成建议关闭思考模式 |
| OSS | `OSS_ENDPOINT`、`OSS_ACCESS_KEY_ID`、`OSS_ACCESS_KEY_SECRET`、`OSS_BUCKET_NAME` | 图片和文件上传 |
| 外部能力 | `DASHSCOPE_API_KEY`、`PEXELS_API_KEY` | 通义与图片搜索能力 |

完整字段与注释见 [后端 `.env` 模板](../backend/coding-server/.env.example)。**不要把 `.env`、密钥、数据库密码或生产地址提交到 Git。**

前端同样应创建本地覆盖文件：

```powershell
Copy-Item frontend/.env.example frontend/.env.local
```

开发时推荐将 `VITE_API_BASE_URL` 设置为 `/api`，由 Vite 代理转发到 `http://localhost:8080`；部署时填写实际 API 地址和应用部署域名。

## 3. 初始化数据库

数据库初始化脚本的文档副本位于：

`doc/sql/init.sql`

它与后端运行资源 `backend/coding-server/src/main/resources/static/sql/init.sql` 内容一致；后者保留不动。创建与 `MYSQL_DATABASE` 一致的数据库后，导入该脚本。启动后端前需确认 MySQL 与 Redis 均可连接。

## 4. 启动后端

在仓库根目录执行：

```powershell
cd backend
./mvnw.cmd -q clean compile
./mvnw.cmd -q -pl coding-server spring-boot:run
```

默认服务地址为 `http://localhost:8080/api`。启动成功后可访问：

- 健康检查：`GET http://localhost:8080/api/health`
- Swagger UI：`http://localhost:8080/api/swagger-ui/index.html`
- OpenAPI JSON：`http://localhost:8080/api/v3/api-docs`

## 5. 启动前端

```powershell
cd frontend
npm install
npm run dev
```

Vite 的 `/api` 代理目标已配置为 `http://localhost:8080`。后端接口有变更时，在后端启动后执行：

```powershell
npm run openapi2ts
```

生成客户端位于 `frontend/src/api/`，请将生成的变更与本次接口变更一起检查。

## 6. 构建与验证

```powershell
# 后端：在 backend 目录
./mvnw.cmd -q test

# 前端：在 frontend 目录
npm run build
```

前端构建会执行 TypeScript 类型检查与 Vite 构建。后端编译成功不等于外部依赖已配置成功；仍应通过健康检查和实际登录/生成流程验证 `.env` 指向的服务。

## 7. 部署前检查

应用部署不是仅启动 Spring Boot：本机还必须有可用的 Nginx，用于提供前端构建产物并反向代理 API、SSE 和已部署应用资源。请在执行页面“部署”操作前，按[部署指南](部署指南.md)完成配置和 `nginx -t` 校验。
