# 接口文档

## 基本约定

- 本地基址：`http://localhost:8080/api`
- 在线调试：`http://localhost:8080/api/swagger-ui/index.html`
- OpenAPI 描述：`http://localhost:8080/api/v3/api-docs`
- 认证：登录后由浏览器携带 Session Cookie；需要登录或管理员角色的接口必须保留 Cookie。
- 普通 JSON 响应：`{ "code": 0, "data": {}, "message": "ok" }`。非零 `code` 表示业务错误。
- 分页请求使用 `pageNum`、`pageSize`，并可附加业务筛选条件；“我的应用”和“精选应用”单页最多 20 条。

前端生成客户端位于 `frontend/src/api/`。接口变更后应以前述 Swagger/OpenAPI 为准并运行 `npm run openapi2ts` 更新客户端。

已部署应用的外部访问地址为 `http://localhost/{deployKey}/`，该地址由 Nginx 反向代理至后端的 `/api/static/{deployKey}/`。详见[部署指南](部署指南.md)。

### SSE 事件格式

`/app/chat/gen/code` 的响应类型为 `text/event-stream`：

- 普通数据帧：`data` 是 JSON 字符串，结构为 `{"d":"生成片段"}`。
- 完成帧：`event: done`。
- 业务错误帧：`event: business-error`，`data` 为 `{"message":"错误信息"}`。

浏览器调用需要保持登录 Cookie；前端应分别处理普通片段、`done` 和 `business-error`。

## 常用调用示例

```bash
# 注册
curl -X POST http://localhost:8080/api/user/register \
  -H "Content-Type: application/json" \
  -d '{"userAccount":"demo_user","userPassword":"password","checkPassword":"password"}'

# 上传图片（先登录并带上 Cookie）
curl -X POST http://localhost:8080/api/file/upload-image \
  -b cookies.txt \
  -F "file=@cover.png"
```

请求字段、返回对象和权限判断可能随源码演进，新增或修改接口后必须同步更新本文件并重新生成前端客户端。
