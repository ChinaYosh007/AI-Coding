# 部署指南

## 1. 部署架构

本项目的“部署应用”流程依赖本机 Nginx：

```text
浏览器
  ├─ /                 -> Nginx 提供 frontend/dist
  ├─ /api/             -> Nginx 反向代理 -> Spring Boot :8080/api/
  └─ /{deployKey}/     -> Nginx 反向代理 -> Spring Boot :8080/api/static/{deployKey}/
```

后端部署接口会将指定应用版本复制到部署目录，并生成 6 位 `deployKey`。后端资源控制器实际提供的是 `/api/static/{deployKey}/...`，而前端展示的部署链接是 `http://localhost/{deployKey}/`，因此没有 Nginx 反向代理时，部署链接无法正确访问。

## 2. 部署前置条件

1. 在 `backend/coding-server/.env` 中配置 MySQL、Redis、模型服务、OSS 等变量。
2. 启动 MySQL、Redis 与后端服务，后端监听 `127.0.0.1:8080`。
3. 在 `frontend/.env.local` 中设置：

   ```dotenv
   VITE_API_BASE_URL=/api
   VITE_DEPLOY_DOMAIN=http://localhost
   ```

4. 在 `frontend` 目录执行 `npm run build`，生成 `frontend/dist`。
5. 安装并启动本机 Nginx。部署操作会访问最终部署 URL 生成截图，因此 Nginx 必须先可用。

## 3. Nginx 配置

将以下 `server` 配置合入 Nginx 配置文件。Windows 路径可使用正斜杠；如 Nginx 安装目录或项目路径不同，请替换 `root` 路径。

```nginx
server {
    listen 80;
    server_name localhost;

    root D:/project/ai-coding/frontend/dist;
    index index.html;

    # 前端构建资源，避免被 6 位 deployKey 正则路由误匹配
    location ^~ /assets/ {
        try_files $uri =404;
        access_log off;
        expires 7d;
    }

    # 后端 API；关闭缓冲以支持代码生成 SSE
    location /api/ {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 3600s;
        add_header X-Accel-Buffering no;
    }

    # 后端生成的 deployKey 为 6 位随机字母数字串。
    # 外部 /{deployKey}/... 映射为后端 /api/static/{deployKey}/...
    location ~ ^/(?<deploy_key>[A-Za-z0-9]{6})(?<deploy_path>/.*)?$ {
        proxy_pass http://127.0.0.1:8080/api/static/$deploy_key$deploy_path;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # Vue Router history fallback
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

> 若访问根域名不是 `localhost`，请同时将 `VITE_DEPLOY_DOMAIN` 设置为相同的公开地址。当前后端部署返回地址由 `AppConstant.CODE_DEPLOY_HOST` 控制，生产环境也应将其调整为相同的公开域名后再构建部署。

## 4. 启动与校验

在 Nginx 安装目录或已配置的命令行环境中执行：

```powershell
nginx -t
nginx -s reload
```

首次启动可执行 `nginx`。完成后依次验证：

```powershell
# API 反向代理
Invoke-WebRequest http://localhost/api/health

# 前端静态站点
Invoke-WebRequest http://localhost/
```

然后登录系统，选择一个已生成的应用版本并执行“部署”。接口返回的 `deployKey` 链接应形如 `http://localhost/Ab12Cd/`，访问该链接应由 Nginx 转发到后端静态资源控制器。

## 5. 常见问题

| 现象 | 排查方向 |
| --- | --- |
| 部署链接 404 | 确认 Nginx 的 `/{deployKey}/` 正则反向代理已生效，后端部署目录中存在对应 key |
| SSE 长时间无输出 | 确认 `/api/` 使用 `proxy_buffering off`，且 `proxy_read_timeout` 足够长 |
| 页面部署成功但封面生成失败 | 先确认 Nginx 与部署 URL 已可从本机访问，再执行部署 |
| 前端请求不到接口 | 确认 `VITE_API_BASE_URL=/api`、Nginx `/api/` 代理和后端 `:8080` 均正常 |
